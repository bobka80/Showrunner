## Categorized problems (priority)

### P0 — Main calendar hover phase strips vanish after prep/timeline

**Item ID:** `E2026-07-20-B`  
**Thread:** [../ERR-2026-07-20-cal-hover-strips.md](../ERR-2026-07-20-cal-hover-strips.md)  
**Kind:** `single`  
**State:** `Open`

**Plain language**  
On the Home month calendar, hovering an event normally reveals the colored **phase strip snapshot** inside the event bar. After you go into prep and/or timeline and come back, that hover snapshot no longer shows (bars look empty / flat).

**Likely files**
- `01a_Calendar_Core.html` — `eventContent` (empty `.bars-wrapper`), `eventDidMount` (fills strips), `calRepaintForkDots_` / `calApplyForkFlagsMap_` / `calOptimisticClearFork_` / fork poll
- `Dal_Firebase.js` / reconcile helpers — Firebase → Sheets (**GAS**) commit on END PREP / END COLLAB (fail-safe B ≈ v711–v712)
- Session close UI (`02e6_Dal_Session.html` / client) — close path that triggers commit + calendar refresh
- `Styles.html` — hover opacity (secondary)

**Hypothesis (ranked)**
1. **Director (lead):** Strips die when **Firebase fails to commit into Google Apps Script (Sheets)** on fork close — not on a clean happy-path leave. Failed/partial close may still run calendar fork-dot clear / contour refresh while phase-bar remount does not re-paint (or event `phases` get stale/wiped).
2. **Mechanism:** Even on that close path, `calRepaintForkDots_` → `setExtendedProp` rebuilds rollup HTML with an **empty** `.bars-wrapper`; strip fill only in `eventDidMount` (≈ fork-dot QoL v698/v704) — so a failed commit’s UI cleanup is enough to trigger the empty bars.
3. **Secondary:** DidMount remounts but DOM geometry early-returns after overlay close.
4. **Unlikely:** CSS hover alone.

**TEST SUGGESTIONS (smoke)**
1. Month view, **before** any prep/timeline: hover a multi-day event → confirm colored phase strips appear on hover.
2. **Clean leave:** Open prep → END PREP successfully → hover same event → strips still present?
3. **Commit-fail leave (director lead):** Force or catch a Firebase→GAS commit failure on close (or note red-dot / restore toast from fail-safe B) → return to calendar → hover → strips gone? Compare to clean leave.
4. Same pair for **timeline** END COLLAB; also wait ~15s fork poll and hover again.
5. Optional DevTools: broken event → `.bars-wrapper` empty vs missing `phases` on the FullCalendar event.
