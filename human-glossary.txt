SHOWRUNNER — HUMAN GLOSSARY
================================================================================
Plain-language terms + one-word names for every screen.
Say the ONE WORD in column 2 when you and the AI (or crew) need the same screen.

Last swept: 2026-06-28 (production UI, GAS v393 era)

================================================================================
PART 1 — SYSTEM WORDS (things you hear in conversation)
================================================================================

PA
  Project Assets. The equipment list for one job/event.
  Where you add gear, pack cases, run the slash formula bar, and SAVE.
  Mobile compact version = same PA, smaller layout.

IAM
  Identity & Access Management. Who can do what.
  In the app this is mostly the ROLE EDITOR (permissions per role)
  and crew login tiers (VIEWER, EDITOR, MANAGER, ROOT).

ROLE EDITOR
  Same as IAM screen. Pick a role, tick permission checkboxes, save.
  Lives inside Resources (ROOT only).

ROOT
  Top permission tier. Full system access — crew, database ops, IAM, audit.

MANAGER / EDITOR / VIEWER
  Login tiers below ROOT. Each tier unlocks different buttons and tabs.

uid
  Unique ID string for a person, project, asset, or row in the database.
  Like a serial number the software uses so names can change without
  breaking links. Example: project_uid separates jobs in one spreadsheet.

Vault
  The master equipment database — every case, fixture, cable type, QR, RFID.
  Not the project list; the warehouse catalog everything is picked from.

Engine (spreadsheet)
  The live Google Sheet backend that stores projects, assets, crew, ledger.
  One Engine file for all projects; rows are filtered by project_uid.

Project Assets vs Vault Assets
  Vault = catalog. Project Assets (PA) = what this job borrowed from catalog.

GAS
  Google Apps Script. The server code that runs Showrunner in Google's cloud.
  When we say "ship" or "deploy", we push a new GAS version.

Firebase / Hosting
  The web address sm-showrunner-97405.web.app — used for push notifications
  and keeping you signed in. The main app UI still loads from GAS.

milestone
  Command: node milestone.js — builds, deploys to GAS, logs version in RELEASES.md.

clasp / build.js
  Developer tools on the PC. build.js bundles HTML into dist/; clasp talks to Google.

Sweep (when talking to AI)
  "Read through the whole codebase/docs for X" — not a button in the app.

Triangle of Truth
  Three things that must stay in sync on a project:
  (1) slash formula you type, (2) pretty formula display, (3) equipment rows.

Formula / CLI bar
  The text bar in PA where you type things like "12x Fixture / Audio".
  Also called pa-search-cli in code.

Beautiful Formula
  Human-readable version of the slash formula for reviewers.

Standalone
  Normal equipment row — not auto-packed, not marked short.

[SHORT]
  Flag on a row meaning we need more than the Vault has available.

Auto-Packing
  Cable/trunk engine. Groups bulk cable into phantom cases ([BULK] … CASE).

Auto-Containerization
  Kit engine. Puts physical fixtures into auto-generated cases (isAuto).
  NOT the same as Auto-Packing.

Bulk
  Countable stock (cables, tape) — quantity, not one RFID per piece.

Container / Case (level 3)
  Physical case with its own QR/RFID. Gear lives inside cases (Matryoshka).

Physical / unique (level 6)
  Individual units (fixtures, mics) — can have RFID each.

Ledger / Operations_Ledger
  Checkout log — what left the warehouse, scans, pack/unpack events.

Session / auto-login
  Stay signed in via token in browser. Desktop can opt out in Personal Hub.
  Different from idle LOCK (screensaver).

Lock (desktop)
  Idle screensaver with clock + PIN. Optional timer in Personal Hub.
  Manual padlock on nav bar works even when timer is Disabled.

Showrunner Sync
  Google Drive shortcuts copied into a project folder (Personal Hub config).

Prep session / Firebase fork (planned, not live)
  Future mode: edit one project in Firebase during prep, then commit to Engine.

Collision (auto-save, planned)
  Someone else saved the same project first — you refresh and retry.

ROOT folder / Drive layout
  Numbered folders on Drive (01_WORKSPACE … 05_DATABASE). See DRIVE_LAYOUT.md.

Fragile zones
  Code areas where a small change breaks PA, timeline, or checkout — touch carefully.

PART 9 — QUICK CHEAT SHEET (most common confusions)
================================================================================

PA              = project equipment list (per job), NOT the vault catalog
Vault / AssetsTab = master catalog of all gear
IAM / RoleEditor = permissions, NOT crew names list
PersonalHub     = your settings, NOT admin Resources
Resources       = admin master data (ROOT/MANAGER), NOT PersonalHub
Tracker         = global "who has what when", NOT the PA list
Timeline        = crew shifts/phases, NOT the month roster
Roster          = crew availability month grid, NOT equipment
Lock            = screensaver PIN, NOT logout
Auto-login      = remember session, NOT lock timer
Auto-Packing    = cables only
Auto-Container  = fixture kits only
uid             = hidden ID; crew "Full Name" can be edited, uid cannot
